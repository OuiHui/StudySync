# Notes — Backend Implementation

## Overview

Notes are the primary content unit in StudySync. A note can be a plain markdown document, a file attachment (PDF, image, etc.), or both. Notes are private by default and can be selectively shared with study groups. In the context of a study group, notes support real-time collaborative presence.

---

## Database Schema

### `notes`

Defined in `20251017000003_clean_and_create_messaging.sql`. Extended by later migrations.

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` | PK, `gen_random_uuid()` |
| `title` | `TEXT NOT NULL` | Required |
| `content` | `TEXT` | Markdown body |
| `subject` | `TEXT` | User-defined category label |
| `group_id` | `UUID → study_groups` | Legacy direct-group assignment (cascades on delete) |
| `created_by` | `UUID → auth.users` | Owner (cascades on delete) |
| `is_collaborative` | `BOOLEAN` | `false` by default; set `true` for group notes |
| `permission_level` | `TEXT` | `'private'` \| `'group'` \| `'public'` \| `'friends'`* |
| `session_id` | `TEXT` | Optional link to a study session |
| `tags` | `TEXT[]` | Unused in current UI |
| `file_url` | `TEXT` | Public URL of an attached file |
| `file_name` | `TEXT` | Original filename for display |
| `created_at` | `TIMESTAMPTZ` | Auto |
| `updated_at` | `TIMESTAMPTZ` | Manually updated on every `updateNote` call |

> *`'friends'` is accepted by the service layer but is **not enforced** by an RLS policy — it currently behaves the same as `'private'`.

### `note_group_shares`

Junction table for the many-to-many relationship between notes and groups. Introduced in `20251031000000_enhance_notes_for_groups_and_subjects.sql`.

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` | PK |
| `note_id` | `UUID → notes` | Cascades on delete |
| `group_id` | `UUID → study_groups` | Cascades on delete |
| `shared_at` | `TIMESTAMPTZ` | Auto |

Unique constraint on `(note_id, group_id)`.

### `custom_subjects`

Per-user label library. Defined in the same migration as `note_group_shares`.

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` | PK |
| `name` | `TEXT NOT NULL` | Unique per user |
| `created_by` | `UUID → auth.users` | Cascades on delete |
| `created_at` | `TIMESTAMPTZ` | Auto |

## Row-Level Security

### `notes` SELECT policy

The final policy (after the recursion fix in `20260706000500_fix_notes_rls_recursion.sql`) allows a user to see a note if **any** of the following is true:

`sql
auth.uid() = created_by
OR public.note_shared_with_user_groups(id)
OR permission_level = 'public'
`

The helper function avoids an RLS cycle (notes <-> note_group_shares both queried each other):

`sql
CREATE OR REPLACE FUNCTION public.note_shared_with_user_groups(p_note_id UUID)
RETURNS BOOLEAN LANGUAGE sql SECURITY DEFINER AS 
  SELECT EXISTS (
    SELECT 1 FROM public.note_group_shares ngs
    JOIN public.group_members gm ON gm.group_id = ngs.group_id
    WHERE ngs.note_id = p_note_id AND gm.user_id = auth.uid()
  );
;
`

### `notes` write policies

| Operation | Enforced by |
|---|---|
| INSERT | `created_by = auth.uid()` (set in service layer) |
| UPDATE | `.eq('created_by', session.user.id)` filter in `updateNote` |
| DELETE | `.eq('created_by', session.user.id)` filter in `deleteNote` |

> Writes are owner-only and enforced at the query level. There is no separate UPDATE/DELETE RLS policy on `notes`.

### `note_group_shares` policies

| Policy | Rule |
|---|---|
| SELECT | User is a member of the `group_id` |
| INSERT | `public.user_owns_note(note_id)` AND user is a member of the `group_id` |
| DELETE | `public.user_owns_note(note_id)` |

`user_owns_note` is a `SECURITY DEFINER` function to bypass the notes table's own RLS during the check.

### `custom_subjects` policies

Standard owner-only CRUD — all four operations gated on `created_by = auth.uid()`.

## Storage

### `note-files` bucket (public)

Used for personal note attachments.

- **Path format:** `{user_id}/{timestamp}.{ext}`
- **Upload:** owner only (folder[1] must match `auth.uid()`)
- **Read:** public (no auth required)
- **Update/Delete:** owner only

### `study_materials` bucket (private)

Used for files uploaded in the context of a group (via `UploadMaterialPopup`).

- **Path format:** `{group_id}/{user_id}/{timestamp}.{ext}`
- **Read:** group members and group creator only
- **Upload:** group members only; restricted to `pdf`, `doc`, `docx`, `txt`, `ppt`, `pptx`, `png`, `jpg`, `jpeg`
- **Update/Delete:** uploader only (folder[2] must match `auth.uid()`)
- Signed URLs (1-hour expiry) are generated by `NotesMutations.getSignedUrl` for private access.

## Service Layer

All database calls go through `src/services/notes/`, which exposes a single facade class.

`
src/services/notes/
├── index.ts       — NotesService facade (re-exports queries + mutations)
├── queries.ts     — NotesQueries: read-only operations
└── mutations.ts   — NotesMutations: writes + file operations
`

### `NotesService` API

#### Queries

| Method | Description |
|---|---|
| `getNotes()` | All notes visible to the current user (own + shared + public), ordered by `updated_at` desc |
| `getNote(id)` | Single note — owner-only (filters `created_by`) |
| `getGroupNotes(groupId)` | Union of `note_group_shares` rows for the group and legacy `group_id = groupId` rows |
| `getSessionNotes(sessionId)` | Notes linked to a study session, ascending by `created_at` |
| `getNoteSharedGroups(noteId)` | Groups a note is currently shared with |
| `getGroupSharedNotes(groupId)` | Notes shared into a group via `note_group_shares` |
| `getUserSubjects()` | Current user's `custom_subjects` rows |

All queries call `populateNoteProfiles` internally, which batch-fetches `profiles` rows and merges `display_name`/`avatar_url` onto each note.

#### Mutations

| Method | Description |
|---|---|
| `createNote(data)` | Inserts a new note; sets `created_by` from the session |
| `updateNote(id, updates)` | Updates own note; always stamps `updated_at` |
| `deleteNote(id)` | Hard deletes own note (cascade removes shares) |
| `uploadFile(file, groupId?)` | Uploads to `study_materials` (if `groupId` given) or `note-files`; returns `{ url, fileName }` |
| `getSignedUrl(filePath)` | Generates a 1-hour signed URL from a `study_materials` path |
| `shareNoteWithGroups(noteId, groupIds)` | Replace-all share: deletes all existing shares, then inserts the new `groupIds` |
| `createSubject(name)` | Inserts a `custom_subjects` row; unique constraint gives a clean error on duplicates |
| `deleteSubject(id)` | Deletes a `custom_subjects` row (owner only) |

## React Data Layer

### `useNotes` (`src/components/notes/useNotes.ts`)

Used by the personal Notes page. Handles:
- Loading notes and groups on mount
- Client-side filtering by search term, subject, and ownership
- CRUD event handlers that call `NotesService` and re-fetch on success
- Sharing flow: `handleShareNote` → `shareNoteWithGroups` + `updateNote(permission_level)`

**Permission auto-sync rule:** when saving group shares:
- If groups were added and `permission_level !== 'group'` → automatically promotes to `'group'`
- If all groups removed and `permission_level === 'group'` → demotes to `'private'`

### `useCollaborativeNotes` (`src/hooks/useCollaborativeNotes.ts`)

Used by `CollaborativeNotes` inside study group pages. Extends the basic service with:

| Feature | Mechanism |
|---|---|
| Real-time note list sync | `RealtimeService.subscribeToNotes({ groupId \| userId })` — Supabase Postgres Changes |
| Online presence | `RealtimeService.trackPresence(groupId, ...)` — Supabase Presence |
| Cursor broadcast | `RealtimeService.broadcast(channel, 'cursor_position', ...)` — Supabase Broadcast |
| Cursor subscription | `RealtimeService.subscribeToBroadcast(channel, 'cursor_position', cb)` |

Cursor positions expire client-side after 5 seconds with no update.

Channel naming:
- Note changes: `group_notes:{groupId}` or `user_notes:{userId}`
- Cursors: `collaborative_notes:{groupId}`
- Presence: `presence:{groupId}`

## Data Flow: Group Note Sharing

`
User selects groups in Share dialog
        |
        v
shareNoteWithGroups(noteId, groupIds)
  1. DELETE FROM note_group_shares WHERE note_id = noteId
  2. INSERT INTO note_group_shares (note_id, group_id) for each groupId
        |
        v
Optionally: updateNote(noteId, { permission_level: 'group' | 'private' })
        |
        v
Other group members can now SELECT the note via RLS:
  note_shared_with_user_groups(id) -> true
`

## Resolved Gaps & Recent Improvements

- **`permission_level = 'friends'`**: Enforced via RLS policy using `public.note_shared_with_friends(created_by)` helper function checking accepted friendships.
- **`group_id` legacy column**: Removed from `notes` table. Note-to-group sharing is exclusively handled via `note_group_shares`.
- **`session_id` referential integrity**: `session_id` column converted to UUID with `FOREIGN KEY (session_id) REFERENCES public.study_sessions(id) ON DELETE SET NULL`.
- **Shared Note Modal**: Unified centered note modal (`SharedNoteModal.tsx`) shared across Notes App, Solo Study, and Group Sessions.
